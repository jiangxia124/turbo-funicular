# 不通过GitHub直接部署到Vercel的方法

本指南将教您如何**不通过GitHub**，直接从本地电脑将贵州大学美食节众筹企划网站部署到Vercel平台。这个方法更加简单直接，特别适合对Git和GitHub不太熟悉的用户。

## 前提条件

在开始之前，请确保您的计算机上已安装以下软件：

- [Node.js](https://nodejs.org/) (建议版本 16.x 或更高)
- [npm](https://www.npmjs.com/) 包管理器 (随Node.js一起安装)

## 步骤1：安装Vercel命令行工具

Vercel提供了一个命令行工具(CLI)，可以让您直接从本地部署项目。请按照以下步骤安装：

1. **打开命令行工具**：
   - **Windows系统**：按下 `Win + R`，输入 `cmd`，然后按回车键
   - **Mac系统**：打开Launchpad，搜索并打开"终端"

2. **安装Vercel CLI**：在命令行中输入以下命令并按回车键：
   ```bash
   npm install -g vercel
   ```
   
3. **等待安装完成**：安装过程可能需要几分钟，取决于您的网络速度

## 步骤2：登录您的Vercel账号

安装完成后，需要登录您的Vercel账号：

1. 在命令行中输入以下命令并按回车键：
   ```bash
   vercel login
   ```

2. 按照提示选择登录方式：
   - 您可以选择使用GitHub、GitLab、Bitbucket账号登录
   - 或者选择使用邮箱登录，Vercel会向您的邮箱发送一个验证链接

3. 完成登录验证后，您将在命令行中看到登录成功的提示

## 步骤3：部署您的项目

现在，您可以直接从本地部署项目了：

1. **进入项目目录**：在命令行中输入以下命令并按回车键（请将路径替换为您实际的项目文件夹路径）：
   ```bash
   cd "您的项目文件夹路径"
   ```
   例如，如果您的项目在`D:\贵州大学美食节众筹企划项目`文件夹中，命令就是：
   ```bash
   cd "D:\贵州大学美食节众筹企划项目"
   ```

2. **初始化并部署项目**：在命令行中输入以下命令并按回车键：
   ```bash
   vercel
   ```

3. **回答Vercel的问题**：系统会问您几个简单的问题，请按照以下提示回答：
   - **Set up and deploy "您的项目路径"?** (是否设置和部署此项目?)：输入 `y` 并按回车
   - **Which scope do you want to deploy to?** (您想部署到哪个范围?)：选择您的Vercel账号，按回车确认
   - **Link to existing project?** (是否链接到现有项目?)：输入 `n` 并按回车
   - **What's your project's name?** (您的项目名称是什么?)：可以使用默认名称，直接按回车
   - **In which directory is your code located?** (您的代码位于哪个目录?)：默认是`.`，直接按回车
   - **Want to override the settings?** (是否要覆盖设置?)：输入 `n` 并按回车

4. **等待部署完成**：Vercel会自动构建并部署您的项目，这个过程可能需要几分钟
   
5. **获取部署链接**：部署完成后，命令行会显示您的项目URL，类似这样：`https://your-project-name.vercel.app`

## 步骤4：查看和管理您的项目

1. 打开浏览器，访问命令行中显示的URL，您的网站应该已经可以正常访问了！

2. 您还可以访问 [Vercel控制台](https://vercel.com/dashboard) 查看和管理您的项目：
   - 查看部署历史
   - 修改项目设置
   - 添加自定义域名（如果需要）
   - 查看访问统计数据

## 更新您的网站

如果您对项目进行了修改，并想更新已部署的网站，只需：

1. 在命令行中进入项目目录
2. 输入以下命令并按回车键：
   ```bash
   vercel --prod
   ```

## 常见问题及解决方案

### 问题1：部署后网站显示白屏或路由不正常

**解决方案**：本项目已经在`vite.config.ts`中优化了配置，确保SPA路由正常工作。如果仍然遇到问题，可以尝试在Vercel控制台中添加一个重写规则：

1. 登录Vercel控制台，选择您的项目
2. 点击"Settings" -> "Domains" -> "Redirects"
3. 添加一条规则：
   - Source: `/*`
   - Destination: `/index.html`
   - Type: `Rewrite`

### 问题2：安装Vercel CLI时出现权限错误

**解决方案**：
- **Windows系统**：以管理员身份运行命令提示符
- **Mac系统**：在命令前加上`sudo`，例如：`sudo npm install -g vercel`

### 问题3：部署过程中出现构建错误

**解决方案**：
1. 确保您的项目可以在本地正常构建：`npm run build`
2. 如果本地构建失败，请先解决构建问题
3. 检查是否有缺失的依赖：`npm install`
4. 查看Vercel控制台中的构建日志，了解具体的错误信息
